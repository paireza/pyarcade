Minesweeper¶

Minesweeper Class¶

class pyarcade.minesweeper_.MinesweeperGame¶

Bases: pyarcade.game_interface.GameInterface

A class representing a Minesweeper game session. A session counter and on_going_session_list will be initialized when the game object is instantiated

All the game session will be stored in the on_going_game_session, with keys including: ‘reference_board’: A answer sheet not public to user. ‘user_board’: A board for user to play ‘mine_list’: mines list ‘win’: A boolean demonstrate if game is finished ‘die’: A boolean demonstrate if user clicked on the mine.

create_game(request: dict) → dict¶

Upon calling create_game, the Minesweeper game should initialize its 9 by 9 game board containing 10 mines denoted as -1, all the numbers around each mines, this one will be used as reference board which will not be seen by user.

Also, we will generate another game board specifically for user to play with. When we update game, we will update based on the user’s move and the reference board.

Here we use 0 to demonstrate empty unit. And we will randomly assign 10 mines on game board denoted as -1.

Args: request: dictionary containing single key-value pair. The key is “game_id”.

Returns: reply: dictionary containing the session_id in the request.

update_game(request: dict) → dict¶
Args:
request: A dictionary contains 3 keys:

“session_id”: session_id provided with the original request. “mode”: A string: there are 2 mode can be used here, one is ‘flag’ mode, one is ‘click’ mode “move”: ‘move’ should be a tuple of 2 integer. showing where the player want to put the piece. The tuple should look like (x, y) where x is integer between [1, 6] and y is integer in between [1, 7].

Returns:

reply: dictionary containing three keys. “moves” contains all the move history player made at current stage. “session_id”: session_id provided with the original request. “win”: True or False depending on whether the game is over.

A class representing a Minesweeper game proxy. Front-end will directly contact with game proxy instead of the game logic itself, game proxy can effectively filter out invalid inputs.

create_game(request: dict) → dict¶

Creating a minesweeper game session. :param request: A dictionary containing the game id. :return: A dictionary containing session id when successful otherwise, return session id of 0.

read_game(request: dict) → dict¶

Reading a existing game session. :param request: A dictionary containing the session id. :return: A dictionary containing 4 keys:

game_board, session_id, winning status and die flag when successful otherwise, return session id of 0.

update_game(request: dict) → dict¶

Updating a minesweeper game session. :param request: A dictionary contains 3 keys:

“session_id”: session_id provided with the original request. “mode”: A string: there are 2 mode can be used here, one is ‘flag’ mode, one is ‘click’ mode. “move”: ‘move’ should be a tuple of 2 integer. showing where the player want to put the piece. The tuple should look like (x, y) where x is integer. between [1, 6] and y is integer in between [1, 7].

Returns

reply: dictionary containing three keys. “moves” contains all the move history player made at current stage. “session_id”: session_id provided with the original request. “win”: True or False depending on whether the game is over.
