class pyarcade.mastermind_.MastermindGame¶

Bases: pyarcade.game_interface.GameInterface

A class representing a Mastermind game session.

Note:

For now, Mastermind must have a hidden sequence of length 4 in which all 4 integers may take on values between 0 and 9.

Upon calling create_game, the Mastermind game should initialize its hidden sequence

Args:

Returns:

reply: dictionary containing the session_id in the request.

read_game(request: dict) → dict¶
Args:

request: dictionary containing single key-value pair. The key is “session_id”. The value is a integer unique to all ongoing game sessions.

Returns:
reply: dictionary containing three keys.

“guesses”: all previous guesses and their respective numbers of cows and bulls for this game_session. All guesses should be kept as a list of tuples under the key “guesses.” A guess of (0, 1, 2, 3) that has one cow and two bulls should be APPENDED to the list as ((0, 1, 2, 3), (1, 2)). “session_id”: session_id provided with the original request. “done”: True or False depending on whether the game is over.

So the overall reply could look like: {“guesses”: [((0, 1, 2, 3), (1, 2), ((3, 2, 1, 0), (2, 1))], “session_id”: 1, “done”: False}

So the overall reply could look like: {“guesses”: [((0, 1, 2, 3), (1, 2), ((3, 2, 1, 0), (2, 1))], “session_id”: 1, “done”: False}

Mastermind Proxy Class¶

class pyarcade.mastermind_proxy.MastermindGameProxy(game_instance: pyarcade.mastermind_.MastermindGame)¶

Bases: pyarcade.game_interface.GameInterface

MastermindGameProxy is meant to be a mediator between client code attempting to play the game Mastermind and the Mastermind implementation. More specifically, this class’s responsibility is to validate requests before passing them on to a Mastermind instance. Separating the responsibility of validating inputs to a game and actually running a game helps us follow the single-responsibility principle

Args:

game_instance: A reference to the game being played.
