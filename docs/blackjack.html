read_game(request: dict) → dict¶

Reads the state of the game.

Args:

request: dictionary containing single key-value pair. The key is “session_id”. The value is a integer unique to all ongoing game sessions.

Returns:
reply: dictionary containing the following key-value pairs:

“session_id”: player’s session_id “player_hand”: cards the player currently has, “total”: the total of user’s current hand, “dealer_hand”: one card that the deal exposes(if the status is in_progress),

otherwise the entire hand.

“status”: “in_progress” when the player is not busted but has not yet stand;
“win”: when the player’s total is 21 while the dealer’s total is not,

player’s total is greater than dealer’s total while not busted, or dealer busts while the player does not;

“lose”: when the player’s total exceeds 21 (busts),

less than the dealer’s total while the dealer is not busted;

“tie”: when the player has the same total as the dealer (including 21)

BlackJack Proxy Class¶

class pyarcade.blackjack_proxy.BlackjackProxy(game_instance: pyarcade.blackjack_.BlackjackGame)¶

Bases: pyarcade.game_interface.GameInterface

BlackjackGameProxy is meant to be a mediator between client code attempting to play the game Blackjack and the Blackjack implementation. More specifically, this class’s responsibility is to validate requests before passing them on to a Blackjack instance. Separating the responsibility of validating inputs to a game and actually running a game helps us follow the single-responsibility principle

Args:

game_instance: A reference to the game being played.
